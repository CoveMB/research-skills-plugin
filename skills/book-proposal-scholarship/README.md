# book-proposal-scholarship

## What it does

Use this skill to develop a research nonfiction book proposal. It helps shape the premise, thesis, contribution, audience, source base, chapter summaries, comparable-title needs, author positioning, manuscript status, timeline, sample chapter strategy, and submission risks.

A proposal describes the book and argues that it has a clear contribution, a real audience, a credible source base, and a workable path to completion.

## When to use it

Use it once the thesis, audience, source base, and chapter structure are stable enough to explain. It can also help early if the author needs to understand what a press, agent, fellowship committee, or grant reviewer will expect.

Use it before contacting presses, agents, editors, or committees. Use it again after major manuscript revisions.

## Good inputs

- Book premise, thesis, audience, and table of contents.
- Chapter summaries or chapter architecture outputs.
- Literature map, source base, or method notes.
- Author bio or credentials, if relevant.
- Intended submission type, such as academic press, trade press, agent, fellowship, grant, or dissertation-to-book revision.

## Example requests

```text
Build a research book proposal from this thesis, chapter outline, and source base.
```

```text
Turn these chapter summaries into proposal-ready chapter descriptions.
```

```text
Identify proposal risks before I send this to an academic press.
```

## Typical output

The output usually includes working title options, one-sentence premise, core thesis, contribution, audience, why now, methods and source base, chapter summaries, comparable-title needs, author positioning, manuscript status and timeline, sample chapter recommendation, risks, and a risk-gated follow-up when it is useful.

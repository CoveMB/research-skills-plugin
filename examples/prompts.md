# Usage examples

## Build the full research workflow

```text
Use research-book-orchestrator. My book argues that [thesis]. Build a full research, writing, and verification workflow.
```

## Route a mixed accessibility bottleneck

```text
Use dyslexia-research-companion. I have rough notes, dictation fragments, and dense source excerpts. Choose the smallest low-load first step and keep ambiguities visible.
```

## Turn dictation into research notes

```text
Use dictation-to-research-notes. Turn this voice transcript into cleaned notes, claims, questions, evidence needs, ambiguities, and next actions.
```

## Reduce reading load

```text
Use reading-load-reducer. Triage these sources into read closely, skim, park, or skip for my current chapter.
```

## Repair prose without changing meaning

```text
Use dyslexia-friendly-prose-editor. Repair spelling, grammar, and sentence boundaries in this passage without changing my argument.
```

## Turn a broad idea into research questions

```text
Use scholarly-research-agenda. Turn this idea into a central research question, subquestions, scope boundaries, and evidence plan: [idea].
```

## Find sources systematically

```text
Use systematic-source-discovery. Build search families, Boolean query strings, database targets, inclusion/exclusion rules, and a search log for: [topic].
```

## Dedupe search results

```text
Use discovery-runner-deduper. Dedupe these exported candidate sources, screen them against my criteria, and draft a search-log update.
```

## Convert annotations into source notes

```text
Use annotation-to-source-note. Turn these document highlights into source-bound notes with quote, paraphrase, metadata, and locator gaps marked.
```

## Build an extraction table

```text
Use extraction-table-builder. Turn these source notes into an extraction table and source matrix before synthesis.
```

## Map the literature

```text
Use literature-review-mapper. Here are my sources and notes. Map schools of thought, debates, consensus, gaps, and how they affect my thesis.
```

## Audit claims

```text
Use claim-evidence-ledger. Extract the major claims in this draft and classify evidence status, citation need, risk, and safer wording.
```

## Trace claims to evidence

```text
Use claim-traceability-graph. Map these claims to source notes, citekeys, locators, evidence status, missing links, and repair actions.
```

## Challenge the thesis

```text
Use counterargument-peer-review. Challenge this thesis like a skeptical peer reviewer, then revise it into a stronger form.
```

## Audit citations

```text
Use citation-integrity-auditor. Check this chapter for unsupported claims, citation-source mismatch, quote/page needs, and fabricated-reference risk.
```

## Verify proposal comps

```text
Use book-comps-verifier. Verify these comparable titles and flag stale, mismatched, fabricated, or unverified positioning claims.
```

## Audit release risk

```text
Use rights-privacy-release-auditor. Check this manuscript export for privacy, copied source text, quote locator, license, and local metadata risks before I share it.
```
